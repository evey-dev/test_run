IMPLEMENTATION AND ANALYSIS CHANGE SUMMARY
==========================================

Purpose
-------

This file records why the implementation changed during the reproducibility
investigation. `report_notes.txt` is the source of truth for the scientific
interpretation; `ai_handoff_summary.txt` contains exact final numbers and pending
work.


1. Contrastive attribution
--------------------------

File:
  src/attribution_graph.py

Added `--contrast-target`. Graph objectives can now be defined as:

  target logit - contrast logit

This was necessary for arithmetic. At the tens digit of 58+83=141, a feature is
relevant if it favours 4 relative to the dropped-carry alternative 3, not merely
if it raises every digit logit. Graph JSON records both tokens and the objective.

The graph renderer now retains JSON, Mermaid and interactive HTML outputs. A
CPU plotting script creates a compact backward-connected report view while
labelling it as a visual subset rather than the complete graph.


2. Intervention correctness and diagnostics
-------------------------------------------

File:
  src/intervention.py

Added or corrected:

- `--graph-feature-sign positive|negative|all`
- `--edit-strength`
- `--knockout-component mlp|attn|block`
- `--positions last|all|comma-separated indices`
- `--layer-scan`
- `--position-scan`
- `--print-tokens`
- `--raw-mlp-swap`
- source activation capture for sparse, complete-latent and raw MLP swaps
- token-position displays and pre-knockout component norms

Reason:

Early outputs did not visibly prove that the intended positions and non-zero
activations were being edited. Printing token IDs, decoded tokens and norms made
position alignment auditable. Error-preserving SAE interventions modify the
true MLP output by only the decoded latent difference, so an empty intervention
is exactly the clean model.

Full MLP, attention and block knockout were retained as component-level
diagnostics. They are intentionally not treated as sparse feature evidence.

A later audit found an exploratory raw-MLP layer-scan path that applied all
requested layers in each nominal row. Those rows are excluded from the report;
aggregate raw patches and all final screens use separate paths. The hook logic
was corrected.


3. SAE checkpointing and diagnostics
------------------------------------

Files:
  src/train.py
  src/sae_diagnostics.py

Training now saves the checkpoint with minimum validation loss rather than the
last epoch. Diagnostics report:

- validation MSE and fraction of variance explained;
- cosine and relative-L2 reconstruction fidelity;
- mean/median/p95 latent L0;
- train, validation and combined dead-feature fractions;
- encoder and decoder norm distributions;
- activation type and TopK metadata.

Reason:

The original weak feature interventions could not be interpreted until SAE
reconstruction and achieved sparsity were measured. The final ReLU diagnosis
was high FVE but thousands of active latents, so simple reconstruction failure
was rejected and density became the next testable problem.


4. Position-aware held-out validation
-------------------------------------

Files:
  src/heldout_validation.py
  run_gpu_final_validation.ipynb

Added deterministic graph-held-out benchmarks with:

- clean source/target eligibility;
- sparse, complete-latent and raw-MLP conditions;
- `--positions` control;
- matched no-carry arithmetic specificity;
- 5,000-resample seeded bootstrap intervals;
- explicit graph-held-out versus SAE-support metadata.

Reason:

One striking prompt can be selected accidentally. Aggregate prompt pairs test
whether the graph-derived edit generalises. Arithmetic also needs a matched
no-carry control; otherwise generic digit-state disruption can look like carry
removal.


5. TopK SAE extension
---------------------

Files:
  src/train.py
  src/attribution_graph.py
  src/intervention.py
  src/sae_diagnostics.py
  src/select_sae_candidate.py

Added backward-compatible TopK SAEs:

  z = TopK_k(ReLU(encoder(x - decoder_bias)))

TopK configurations use unit-normalised decoder columns and `l1_lambda: 0.0`.
Original ReLU configurations retain their old behaviour.

Fixed candidate values are k=128, 256 and 512. Candidate selection is isolated
from intervention outcomes. Required thresholds:

  mean validation FVE >= 0.90
  minimum layer validation FVE >= 0.85
  mean combined dead fraction <= 0.80

Among eligible candidates, choose the smallest mean validation L0.

Reason:

Hard TopK directly controls support size and unit decoder norms remove the
scale ambiguity of the original raw-L1 objective. The intervention must then be
tested again; improved sparsity alone does not prove mechanistic specificity.


6. Arithmetic TopK specificity screen
-------------------------------------

Files:
  configs/sae_math_topk128_config.yaml
  configs/sae_math_topk256_config.yaml
  configs/sae_math_topk512_config.yaml
  src/math_carry_feature_screen.py
  src/plot_topk_math.py
  src/plot_math_carry_screen.py
  run_gpu_math_topk_retrain.ipynb
  run_gpu_math_carry_feature_followup.ipynb

The metric-only rule selected TopK-256. It increased the all-position sparse
carry-target effect from the original ReLU mean -0.396 to -0.875 logits.
However, the no-carry control changed by -0.885, so the effect was not
carry-specific.

The final-position feature screen:

- excluded previously inspected benchmark cases;
- ranked individual graph features on 8 discovery pairs;
- froze the top-10 panel;
- tested 24 disjoint confirmation pairs;
- retained bottom-ranked and five matched random controls.

The primary paired interval crossed zero. This is a predeclared negative result
and is retained in the report.

An output-digit-balanced extension was then completed in
`run_gpu_math_carry_balanced_localization.ipynb`. It ranked all 57,344
TopK-256 latents on 32 discovery pairs while conditioning on the tens digit,
then tested frozen panels on 32 disjoint confirmation pairs. The Top-10 panel
was perfectly decodable on confirmation activations but failed causal
specificity. A secondary Top-20 panel produced a negative paired interval.

Because Top-20 was secondary, `src/math_carry_top20_replication.py` and a final
notebook section now perform one independent replication of those exact 20
feature IDs on intervention-untouched eligible cases. They do not rerank or
search another panel. The stopping rule is explicit so that this follow-up can
upgrade a hypothesis-generating result without silently replacing the failed
primary analysis.

The frozen replication passed on 32 untouched cases: carry target -0.0586,
no-carry control +0.0273, and paired specificity -0.0859 with bootstrap 95%
interval [-0.1367, -0.0391]. No top prediction changed. Panel search stops at
this result. The report describes a distributed carry-associated causal panel
under one SAE basis, not a single carry feature or complete circuit.


7. Units TopK specificity screen
-------------------------------

Files:
  configs/sae_units_topk128_config.yaml
  configs/sae_units_topk256_config.yaml
  configs/sae_units_topk512_config.yaml
  src/units_feature_screen.py
  src/plot_topk_units.py
  run_gpu_units_topk_retrain.ipynb

The metric-only rule selected TopK-128. The final screen:

- took 73 positive-attribution candidates from one force graph;
- ranked features on 8 discovery physical systems;
- used force-source minus matched mass-source effect as the estimand;
- froze a top-10 panel;
- tested 16 different physical systems;
- ensured exact prompts were absent from the SAE corpus;
- used one prompt per system and disjoint system groups;
- retained cumulative, reverse-ranked, random, layer, all-graph, full-latent and
  raw-MLP comparators.

The original indirect prompt wording yielded too few fully qualified systems.
A baseline-only audit was therefore changed to direct SI-unit wording before any
feature intervention. Four wording variants over 64 systems were checked. This
protocol amendment is recorded in output JSON and is not post-outcome selection.

The frozen top-10 panel passed its predeclared rule on 16/16 systems:

  force shift       +1.2344 [1.1484, 1.3320]
  mass shift        -0.0898 [-0.1289, -0.0508]
  paired effect     +1.3242 [1.2227, 1.4297]

No top prediction changed. The implementation and report call this a compact
force-associated panel, not a complete circuit.

chronological notebook log. Exploratory broad controls lead to measured SAE
density, then to fixed TopK selection, matched specificity tests and the final
units confirmation.


9. Colab, GitHub and Drive execution
-----------------------------------

Colab notebooks pull the code repository from GitHub first. A Drive zip remains
as backup. Large activation arrays, checkpoints and outputs are copied to Google
Drive; existing `!cp` and `!rm` lines requested by the user were retained.

Colab was used because CSD3 became unavailable during a late-June cooling
incident associated with hot weather. The project lost unbacked activation and
checkpoint artifacts and regenerated them. Final recorded runtime hardware
includes NVIDIA T4 and A100-SXM4-80GB.

Do not state a fixed Colab compute-unit rate. Google's FAQ says resource limits,
GPU availability and usage conditions vary and are not published as fixed
limits. The report describes a compute-unit budget without using the guessed
6.77 units/hour figure.
