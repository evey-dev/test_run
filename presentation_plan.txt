========================================================================
PRESENTATION PLAN
Tracing Sparse Causal Features in an Open Language Model
A Small-Scale Reproducibility Study of Attribution Graphs
========================================================================

Format:        Short conference talk, 25 min + 10 min questions
Audience:      Scientifically literate cohort peers. Familiar with core
               MPhil modules (ML, statistics, data analysis), NOT with
               mechanistic interpretability specifically.
Target length: ~22 content slides (~1 min each), leaving buffer + Q&A.
               Backup slides at the end for anticipated questions.

Guiding principle for delivery:
  Slides carry ONE visual idea each. Most of the content lives in the
  figures and in what you SAY, not in bullet text. Bullets below marked
  "SLIDE TEXT" are the maximum that should appear; "SPEAK" is your script.

Narrative arc (keep this in mind for cohesion):
  Motivation -> what the target paper did -> can it travel to an open
  model? -> pipeline -> the pivotal discovery that reconstruction is not
  sparsity -> the fix (TopK) -> one strong positive (units), one honest
  mixed result (arithmetic), one clean null (capitals) -> what this
  teaches us about the method -> limits -> next steps.

The intellectual spine is a single sentence, repeat it 2-3 times:
  "High reconstruction fidelity, held-out decodability, and sparse causal
   control are three DIFFERENT things, and only the last one supports a
   mechanistic claim."

SLIDE 7 — Step 1: Sparse autoencoders (how they work)
------------------------------------------------------------------------
TITLE:
  Recovering interpretable features with SAEs

SLIDE TEXT (max):
  - Encode MLP activation u -> sparse code z (ReLU), decode -> u-hat
  - Overcomplete (8192 latents > 2560 dims): sparsity, not compression
  - Quality = Fraction of Variance Explained (FVE); FVE=1 is perfect

FIGURE/DIAGRAM:
  Custom schematic (create): u (dense vector, 2560) -> encoder -> z
  (tall sparse bar chart, mostly zeros, 8192) -> decoder -> u-hat
  (dense, 2560). Annotate "each active coordinate = a learned direction /
  candidate feature." Show the two key equations (ReLU-SAE + FVE)
  compactly beside it.

SPEAK:
  - Explain the autoencoder plainly for peers: reconstruct the activation
    from a few active learned directions; those directions are our
    candidate features.
  - Define FVE quickly (variance-explained, like R^2) so the diagnostics
    later land.
  - IMPORTANT foreshadow: say out loud "high FVE means good
    reconstruction — hold that thought, because it will NOT mean what you
    expect." Plants the pivotal result.
  - Training detail worth one sentence: independent per-layer SAEs at
    layers {4,8,12,16,20,24,28}, 800 train / 200 val vectors each.


------------------------------------------------------------------------
SLIDE 8 — Step 2: Attribution graphs (how I build them)
------------------------------------------------------------------------
TITLE:
  From features to a causal hypothesis: the contrast graph

SLIDE TEXT:
  - Objective J = logit(target) - logit(contrast)   e.g. logit(4)-logit(3)
  - Rank features by first-order attribution; keep top nodes per layer
  - Straight-through splice: graph gradients WITHOUT changing the output

FIGURE/DIAGRAM:
  report/figures/fig_math_attribution_graph.pdf  (the arithmetic graph,
  logit(4) vs logit(3)). Red/blue = positive/negative attribution.
  Note in caption: this is a backward-connected visual SUBSET.

SPEAK:
  - Explain the contrast objective concretely with 58+83: we score how
    much each feature pushes "4" (correct, carry) over "3" (dropped carry).
  - One design-rationale sentence on the straight-through splice: it lets
    me attribute gradients through the SAE while keeping the model's
    actual forward pass unchanged — so reconstruction error doesn't
    contaminate the baseline. (This is a deliberate, defensible design
    choice — good for the "rationale for design choices" criterion.)
  - Reiterate: the graph is still just a hypothesis. Next slide: the test.


------------------------------------------------------------------------
SLIDE 9 — Step 3: Interventions + the discipline that makes them credible
------------------------------------------------------------------------
TITLE:
  Testing the hypothesis: interventions & pre-registration

SLIDE TEXT:
  - Inhibition: remove selected features' contribution to the MLP output
  - Swap: inject a donor prompt's feature values into the target
  - Discovery/confirmation split; panels FROZEN before confirmation
  - Matched control decides specificity (no-carry / mass donor)

FIGURE/DIAGRAM:
  report/figures/fig_intervention_schematic.pdf
  (clean forward pass vs. same prompt after editing only the frozen
  panel latents). This is the cleanest single picture of "what an
  intervention is."

SPEAK:
  - Two intervention types in plain terms: turn features off (inhibition),
    or paste features from another prompt (swap).
  - THE methodological backbone — spell it out slowly, this earns depth
    marks: features are ranked on a DISCOVERY set, the panel is frozen,
    then tested ONCE on a disjoint CONFIRMATION set. Confirmation data
    never influence which features are chosen.
  - Matched controls define specificity: for carry, a no-carry pair that
    should NOT move if the feature is truly carry-specific; for units, a
    mass donor that should NOT transfer a "newtons" preference.
  - Success criteria were pre-declared (sign + bootstrap interval wholly
    past zero). This is what separates a real effect from a story.

------------------------------------------------------------------------
SLIDE 13 — The fix: fixed-sparsity TopK SAEs
------------------------------------------------------------------------
TITLE:
  The correction: TopK sparse autoencoders

SLIDE TEXT:
  - Replace L1 penalty (scale-degenerate) with hard TopK: keep k active
  - Pre-declared candidates k in {128, 256, 512}; decoder norms fixed to 1
  - Selection rule set BEFORE seeing any intervention result

FIGURE/DIAGRAM:
  Custom small diagram (create): the same tall sparse-code bar from
  slide 7, but now with a horizontal "keep top-k" cut line — everything
  below is zeroed. Side note box: selection thresholds (FVE>=0.90,
  min-layer FVE>=0.85, dead-fraction<=0.80; pick sparsest eligible).

SPEAK:
  - Design rationale (depth marks): the L1 objective is scale-degenerate —
    you can rescale encoder/decoder and cheat the penalty, so it doesn't
    truly control sparsity. TopK controls the support SIZE directly.
  - Stress the pre-registration again: candidate ks and the selection
    thresholds were fixed before any causal outcome was seen. Selection
    used only reconstruction/sparsity diagnostics.
  - This is the same fix applied identically to arithmetic and units —
    enabling a fair comparison between them.

BACKUP / APPENDIX SLIDES (for the 10-min Q&A — do not present)
========================================================================

B1 — SAE training details
  Loss (MSE + L1), 75 epochs, Adam lr 1e-3, batch 64, 80:20 split,
  ~42M params/layer, activation normalisation s_ell. For "how exactly did
  you train them?"

B2 — TopK selection numbers
  Arithmetic: TopK-128 dead-fraction 0.832 (FAILED threshold) -> TopK-256
  chosen (FVE 0.941, L0 171.6). Units: TopK-128 chosen (FVE 0.936,
  L0 99.8). For "why 256 for math but 128 for units?"

B3 — Bootstrap / statistics
  5,000 seeded resamples; intervals are 2.5/97.5 percentiles; quantify
  PROMPT-SYSTEM variation, condition on the evaluation pool, do NOT capture
  SAE-seed uncertainty. For "how robust are the intervals?"

B4 — The straight-through splice & intervention algebra
