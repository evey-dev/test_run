==================

Purpose
-------
This file is the source of truth for every main-report figure. Use vector PDF
files in report/report.tex; PNG files are previews only. The existing plots and
pipeline diagram are generated from code, so no manual redrawing is required.
The interactive graph HTML files remain useful supplementary artifacts but are
too dense to serve as primary report figures.

Status key
----------
INSERTED: generated, checked in the compiled report, and already referenced.
PENDING RUN: code exists, but the final GPU notebook must generate real values.
PENDING INSERT: output exists only after that run and must then be copied into
report/figures and referenced from report.tex.

6. SAE reconstruction and sparsity diagnostics
-----------------------------------------------
Exact filename: report/figures/fig_sae_diagnostics.pdf
Generator: src/plot_validation.py
Source artifacts:
- outputs/final_sae_diagnostics_math.csv
- outputs/final_sae_diagnostics_units.csv
- outputs/final_sae_diagnostics_capitals.csv
Status: PENDING RUN, then PENDING INSERT.
Location: Results, in a new subsection immediately after baseline competence.
Suggested label: fig:sae-diagnostics.

Content:
- held-out validation fraction of variance explained (FVE) by layer;
- mean active latent count (L0) by layer;
- dead-feature fraction by layer;
- separate curves for addition, SI units and capitals.

Suggested caption:
  Layerwise SAE fidelity and achieved sparsity on the original deterministic
  validation splits. FVE quantifies reconstruction relative to prediction by
  the training-set mean; L0 is the mean number of non-zero latents, and the
  dead-feature fraction is measured over the combined training and validation
  activation support.

Interpretation rule:
- Low FVE or large relative error supports poor reconstruction fidelity as one
  plausible contributor to weak sparse interventions.
- High FVE with weak sparse interventions rules out that simple explanation and
  shifts attention to achieved sparsity, feature instability, graph selection,
  support mismatch and nonlinear feature interactions.
- These are diagnostic associations. They do not prove that reconstruction
  error caused an intervention failure.

Required report edit after the run:
The current Discussion sentence saying that final L0, dead-feature and related
diagnostics were not retained must be replaced with the measured values. Do not
retain that sentence once this figure is inserted.

How to generate all pending figures
-----------------------------------
Run run_gpu_final_validation.ipynb from top to bottom. It does not retrain an SAE
or construct a new attribution graph. It restores the existing final artifacts,
recaptures deterministic activation matrices only if absent, evaluates the
checkpoints, runs 12 arithmetic and 12 unit prompt-pair interventions, generates
the three pending figures and copies outputs to Drive.

Expected runtime is approximately 10--20 minutes on a Colab GPU, depending on
model loading and Drive transfer. It should be substantially shorter than the
attribution-graph cells. The graph-held-out benchmark is the only GPU-intensive
part; figure generation itself is CPU-only and fast.
