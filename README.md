# Mechanistic Interpretability Reproduction Project

This repository reproduces a small-scale mechanistic interpretability workflow for Qwen3-4B-Instruct using three behaviors: capitals, addition, and physics units.

## Environment & Workspace setup
Create and activate the project environment:

```bash
conda env create -f environment.yml
conda activate mphil-project
```

### Prerequisites and expected files
- [setup.py](setup.py) is optional package metadata for the project. It is useful if you want to install the repository with `pip install -e .`, but it does not download the model or generate the datasets.
- The baseline and training scripts assume that the model weights already exist locally. The default path is [configs/model_config.yaml](configs/model_config.yaml), which points at [models/Qwen3-4B-Instruct](models/Qwen3-4B-Instruct). If that folder is missing, download or restore the model into that location first, for example:

```bash
huggingface-cli download Qwen/Qwen3-4B-Instruct --local-dir models/Qwen3-4B-Instruct
```

- The repository also expects a few generated or precomputed artifacts:
  - Prompt CSVs: generated by the dataset script below.
  - Baseline JSON outputs: generated by the baseline script below.
  - SAE checkpoints and latents: generated by the training script below.
  - Activation tensors and split indices: generated by the activation capture script below.

### Generate the prompt datasets
Run the dataset generator from the repository root:

```bash
python data/generate_datasets.py --capitals
```

Optional flags:
- `--capitals`: also generate the geography prompt CSV from the base capitals file.

The script writes the prompt CSV files into the data directory:
- data/addition_data.csv
- data/units_data.csv
- data/capitals_data.csv

## Capture real model activations
Extract the hidden activations (MLP block outputs) of the local language model on the prompt datasets. Run the capture script from the repository root:

```bash
python src/capture_activations.py --output-dir mechanistic_data --seed 787
```

Optional flags:
- `--layers`: override which layers to capture (default: 8, 16, 24, 32)
- `--behaviours`: override which behaviours/prompts to capture (default: capitals, addition, units)
- `--copy-model`: save a checkpoint copy of the local model into the output directory
- `--model-config`: path to the model config file (default: configs/model_config.yaml)

This writes the activation files into [mechanistic_data/](mechanistic_data/), including:
- activation tensors for each requested layer (`activations_layer*.npy`)
- train/validation split arrays (`train_indices.npy`, `val_indices.npy`, `train_val_indices_per_layer.npy`)
- activation metadata CSV (`activation_metadata.csv`)

## 1. Run the baseline evaluation
Baseline inference uses the prompt CSV files above and writes outputs to the outputs/baselines directory.

```bash
python src/baseline.py --config configs/baseline_config.yaml --prefix 10
```

Optional flags:
- `--model-config`: override the model configuration file
- `--config`: override the baseline configuration file
- `--prefix`: set the number of prefix-guided continuation attempts for capitals/units (0 disables this behavior)

Outputs:
- outputs/baselines/all_baselines.json
- outputs/baselines/capitals_baselines.json
- outputs/baselines/addition_baselines.json
- outputs/baselines/units_baselines.json

These files contain the model’s top-k predictions, expected-answer ranks, probabilities, and prompt metadata for each behavior.

## 2. Train the sparse autoencoders (SAEs)
Training uses the real activations and split indices generated in the [mechanistic_data](mechanistic_data) directory. If the activation files are not found, `train.py` will automatically invoke the activation capture script on-the-fly to capture them before starting training.

```bash
python src/train.py --config configs/sae_config.yaml
```

Optional flags:
- `--layers`: override the layers to train
- `--epochs`: override the number of training epochs
- `--batch-size`: override the training batch size
- `--latent-dim`: override the latent dimension

Optional smaller smoke test:

```bash
python src/train.py --config configs/sae_config.yaml --layers 8 --epochs 1 --batch-size 8 --latent-dim 64
```

Outputs:
- mechanistic_data/sae_checkpoints/sae_layer8.pt
- mechanistic_data/sae_checkpoints/sae_layer16.pt
- mechanistic_data/sae_checkpoints/sae_layer24.pt
- mechanistic_data/sae_checkpoints/sae_layer32.pt
- mechanistic_data/sae_checkpoints/latents_layer8.npy
- mechanistic_data/sae_checkpoints/latents_layer16.npy
- mechanistic_data/sae_checkpoints/latents_layer24.npy
- mechanistic_data/sae_checkpoints/latents_layer32.npy
- mechanistic_data/sae_checkpoints/sae_layer*_metadata.json

These files contain the trained SAE weights, projected latent activations, and basic training metadata for each layer.

## 3. Construct attribution graphs (Phase 5)
Construct a pruned causal dependency graph from input tokens through SAE latent features to the target logit. The graph is optimized to run efficiently on CPU (disabling parameter gradients and only computing Jacobians for active features) and performs automatic leading-space token matching.

```bash
python src/attribution_graph.py --prompt "Fact: the capital of the state containing Dallas is" --target "Austin" --top-k-nodes 5 --top-k-edges 5
```

Optional flags:
- `--target`: specify next token to track (defaults to model's top prediction). Automatically matches raw, leading-space, or stripped forms.
- `--top-k-nodes`: number of top nodes to keep per layer (default: 10)
- `--top-k-edges`: number of top edges to keep per layer (default: 15)
- `--output-json`: path to save the JSON output graph (default: `outputs/attribution_graph.json`)
- `--output-html`: path to save interactive Vis.js HTML visualization (default: `outputs/attribution_graph.html`)
- `--output-mermaid`: path to save Mermaid flowchart markdown (default: `outputs/attribution_graph.md`)

Outputs:
- A structured JSON representation of the pruned graph.
- An interactive Vis.js HTML file enabling graphical node/edge rendering (zoom, pan, drag) in the browser.
- A Markdown file with Mermaid code representing the flowchart.

## 4. Run intervention experiments (Phase 6)
Perform causal interventions (inhibition or activation swap-in) on SAE features to validate attribution paths.

### Inhibition (Ablation)
Zero out specified features during forward pass and observe changes:
```bash
python src/intervention.py --mode inhibit --prompt "Fact: the capital of the state containing Dallas is" --features '{"16": [238]}' --target-token "Austin"
```

### Activation Swap-In
Capture feature activations from a source prompt (e.g. Oakland -> Sacramento) and inject/swap them into a target run (e.g. Dallas -> Austin) to measure logit shifts or output flips:
```bash
python src/intervention.py --mode swap --source-prompt "Fact: the capital of the state containing Oakland is" --prompt "Fact: the capital of the state containing Dallas is" --features '{"16": [492]}' --target-token "Sacramento"
```

Optional flags:
- `--features`: JSON string specifying which features to intervene on, e.g. `'{"layer": [idx1, idx2]}'` (if omitted in swap mode, the entire layer is swapped).
- `--target-token`: target next token to track probability and logit shifts.

## Notes on data usage
- The baseline script expects the prompt CSV files in the data directory.
- The training script expects the activation tensors and split indices in mechanistic_data/.
- If the CSVs or activation files are missing, the scripts will raise a clear file-not-found error.
